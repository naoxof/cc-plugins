# tmplt

Markdownファイル内の`<CLAUDE>`プレースホルダーをインタラクティブに補完するClaude Codeプラグイン。

## Features

- プレースホルダー補完の自動化
  - 技術情報は自動収集、ビジネス判断のみユーザーにヒアリング
  - 周辺コンテキストの分析による適切なコンテンツ生成
- 選択肢ベースのインタラクティブなヒアリング
- テンプレート自動生成機能（オプション）

## Usage

### filler: プレースホルダーの補完

Markdownファイルに`<CLAUDE>`プレースホルダーを配置:

```markdown
# プロジェクト提案書

## 概要
<CLAUDE: プロジェクトの目的と期待される成果を簡潔に記述>

## 背景
<CLAUDE: 現在の課題とこのプロジェクトが解決する問題>

## 提案内容
<CLAUDE: 具体的なアプローチを箇条書きで説明>

## 次のステップ
<CLAUDE: 承認後のアクションアイテム>
```

fillerスキルを実行:

```bash
/tmplt:filler proposal.md
```

処理フロー:
1. 周辺コンテキストの分析
2. 技術情報の自動収集（コードベース検索、Web検索）
3. ユーザーへのヒアリング（選択肢ベース）
4. コンテンツ起草
5. プレースホルダー置換

### creator: テンプレートの生成

`<CLAUDE>`プレースホルダーを含むテンプレートファイルを自動生成:

```bash
/tmplt:creator proposal.md
```

要件ヒアリング:
- ドキュメントの種類・目的
- 対象読者
- 必要なセクション・構成
- スタイルとトーン

生成されたテンプレートを必要に応じて編集後、fillerで補完。

## Examples

### 構造化ドキュメントの作成

プロジェクト提案書を作成する例:

```bash
/tmplt:creator project-proposal.md
```

creatorがヒアリング:
- ドキュメント種類（新規プロジェクト、予算申請など）
- 対象読者（経営層、部門長など）
- 必要な要素（予算、スケジュール、リスク分析など）

生成されたテンプレートを必要に応じて編集後、fillerで補完:

```bash
/tmplt:filler project-proposal.md
```

fillerは各プレースホルダーについて選択肢を提示:

```
質問: このプロジェクトの主な目標は？
  A) コスト削減（運用コストを20-30%削減）
  B) 売上増加（新規顧客獲得や既存顧客単価向上）
  C) 業務効率化（作業時間の短縮や自動化）
  D) リスク低減（セキュリティやコンプライアンス対応）
```

ユーザーは選択肢から選ぶか、自由記述で回答。fillerが回答を基にコンテンツを起草し、プレースホルダーを置換。

## Placeholder Syntax

### 基本形式
```markdown
<CLAUDE>
```
周辺コンテキストから内容を推測。

### ヒント付き
```markdown
<CLAUDE: ここにユーザーの目標や成功基準を記述>
```
補完すべき内容を明示的に指定。

## Technical Details

### インタラクティブヒアリング
`AskUserQuestion`ツールによる選択肢ベースの対話。自由記述での回答も可能。

### 自動情報収集
- コードベース検索
- 既存パターンの分析
- Web検索による最新情報の取得

ビジネス判断や好みなど、ユーザー固有の情報のみをヒアリング。

### トレンド対応
技術スタック、製品などトレンドが変わる分野では、Web検索で最新情報を収集してから選択肢を生成。

## Best Practices

### テンプレート設計
- 段落や論理的なセクション単位でプレースホルダーを配置
- プレースホルダーの前後にコンテキストを提供
- ヒントは具体的に（「3つの主要な機能と各々のメリットを記述」など）
- 見出しレベルで階層構造を明確化

### テンプレート利用
- プレースホルダーを1つずつ確認しながら段階的に補完
- 起草された内容をレビューし、必要に応じて修正を依頼
- 利用後の改善点を次回のテンプレートに反映

- ビジネスドキュメント: 提案書、報告書、計画書

- その他、構造化されたMarkdownドキュメント

## FAQ

### Q: テンプレートは自分で作る必要がありますか？
A: 手動作成も可能ですし、creatorスキルで自動生成することも可能です。`<CLAUDE>`を含むMarkdownファイルであれば使用できます。

### Q: 既存のMarkdownファイルにも使えますか？
A: 既存ファイルに`<CLAUDE>`を挿入することで、その部分のみを補完できます。

### Q: プレースホルダーをスキップできますか？
A: fillerスキルにスキップを指示することで、後回しにできます。

### Q: 起草内容の修正は可能ですか？
A: 補完時に修正を依頼することで、即座に内容を修正できます。

### Q: ヒントは必須ですか？
A: 推奨ですが必須ではありません。`<CLAUDE: 具体的な指示>`形式でヒントを付けるとより適切なコンテンツが生成されます。

### Q: テンプレートの再利用は可能ですか？
A: 一度作成したテンプレートは何度でも利用できます。
